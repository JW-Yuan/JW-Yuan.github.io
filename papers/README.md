# 论文阅读心得文件说明

## 文件结构

```
papers/
├── [paper-id]/                   # 每篇论文的文件夹 (例如: 0001/)
│   ├── info.json                 # 论文基本信息文件
│   └── detail.md                 # 论文详细信息（Markdown 格式）
```

所有论文索引信息存储在根目录的 `papers.json` 文件中。

## 论文基本信息文件格式

每篇论文的基本信息文件（`info.json`）应包含以下字段：

```json
{
    "id": "0001",                     // 论文唯一标识符（必需，4位数字ID）
    "name": "论文完整标题",            // 论文完整名称（必需）
    "shortName": "论文缩写",          // 论文缩写名称（可选，如果提供则在列表页显示缩写，全称显示在其他信息列）
    "year": 2024,                     // 发表年份
    "task": ["segmentation", "classification"],  // 任务类型 (数组格式)
    "authors": "作者1, 作者2, 作者3", // 作者列表
    "venue": "会议/期刊名称",          // 发表会议或期刊
    "summary": "论文核心思想总结",    // 核心思想
    "insights": "个人阅读心得",        // 个人心得
    "links": {                        // 相关链接
        "pdf": "https://...",
        "code": "https://..."
    }
}
```

**重要**：`id` 字段必须与文件夹名称一致，且为 4 位数字。`task` 字段应为字符串数组。

## 论文详细信息文件格式

详细信息文件应放在每篇论文的文件夹中，使用 **Markdown 格式**，文件名固定为 `detail.md`。

### Markdown 文件示例

```markdown
# 论文标题

## 论文信息

- **标题**：论文标题
- **年份**：2024
- **作者**：作者1, 作者2, 作者3
- **会议/期刊**：会议/期刊名称
- **任务类型**：分割、分类

## 核心思想

这里是论文的核心思想总结。简要描述论文的主要贡献和创新点。

### 主要贡献

1. 贡献点 1
2. 贡献点 2
3. 贡献点 3

### 创新点

- 创新点 1
- 创新点 2

## 方法概述

详细描述论文提出的方法...

## 实验设计

描述实验设置和数据集...

## 个人心得

这里记录你的阅读心得、思考、启发或对该论文的评价。可以包括方法的优缺点、实验设计的思考等。

### 优点

- 优点 1
- 优点 2

### 缺点

- 缺点 1
- 缺点 2

### 启发

记录这篇论文对你的启发和思考...

## 相关资源

- [PDF 链接](#)
- [代码链接](#)
- [项目主页](#)
```

### Markdown 支持的功能

详情页面支持标准的 Markdown 语法，包括：

- **标题**：`# H1`, `## H2`, `### H3` 等
- **列表**：有序列表和无序列表
- **代码块**：使用三个反引号包裹代码
- **行内代码**：使用单个反引号
- **链接**：`[链接文本](URL)`
- **图片**：`![alt文本](图片路径)`
- **表格**：标准 Markdown 表格语法
- **引用**：使用 `>` 符号
- **粗体**：`**粗体文本**`
- **斜体**：`*斜体文本*`

## 添加新论文

1. **创建论文文件夹**：在 `papers/` 文件夹中创建新的 4 位数字 ID 文件夹（如 `0004/`）
2. **创建基本信息文件**：在该文件夹中创建 `info.json`
3. **创建详细信息文件**（可选）：在该文件夹中创建 `detail.md`
4. **添加到索引**：在根目录的 `papers.json` 中添加新的 ID 和论文名称

### 示例

创建 `0004/info.json`:
```json
{
    "id": "0004",
    "name": "New Paper - Complete Long Title Here",
    "shortName": "New Paper",
    "year": 2024,
    "task": ["detection"],
    "authors": "Author1, Author2",
    "venue": "Conference Name",
    "summary": "Paper summary...",
    "insights": "Personal insights...",
    "links": {
        "pdf": "https://example.com/paper.pdf",
        "code": "https://github.com/example/code"
    }
}
```

然后在根目录的 `papers.json` 中添加：
```json
{
    "0001": "示例论文 1",
    "0002": "示例论文 2",
    "0003": "示例论文 3",
    "0004": "New Paper"
}
```

## 注意事项

- 文件夹名和 `info.json` 中的 `id` 字段必须是唯一的 4 位数字 ID
- 所有链接字段都是可选的
- `year` 字段用于排序和筛选，建议填写
- `task` 字段用于筛选，建议填写
- 索引文件 `papers.json` 位于网站根目录，格式为字典：`{ "0001": "论文名称", ... }`

## 优势

使用 Markdown 格式的优势：

- ✅ **易于编写**：使用纯文本格式，无需学习 JSON 语法
- ✅ **格式丰富**：支持标题、列表、表格、代码块等多种格式
- ✅ **易于维护**：可以直接在 GitHub 上编辑和预览
- ✅ **版本控制友好**：Markdown 文件在 Git 中更容易查看差异
- ✅ **可读性强**：即使不渲染，Markdown 文件也具有良好的可读性
